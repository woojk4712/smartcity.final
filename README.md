# 판교 1테크노밸리 · 청라국제업무지구 비교 GIS

React + Vite + Leaflet 기반 정적 GIS입니다. 업로드된 소재지 엑셀과 연속지적도를 PNU로 매칭해 판교와 청라의 분석 경계를 생성하고, SGIS 2023 CSV와 2023년 말 기준 수도권 철도망을 사용해 비교 화면을 구성합니다.

본 시스템은 계획상 용지 기준이 아니라 실제 업무기능이 형성된 지역을 기준으로 업무지구를 정의하였다.

인구가구사업체종사자 데이터가 2023년 기준이므로, 교통 접근성 분석 역시 2023년 말 기준 운영 중인 수도권 철도망으로 통일하였다.

## 실행

```bash
npm install
npm run preprocess
npm run dev
npm run build
```

## 데이터 구조

- `public/data/boundaries/*_boundary.geojson`: 소재지 PNU와 연속지적도 매칭 후 병합한 구역계
- `public/data/parcels/*_matched_parcels.geojson`: 구역계 생성에 사용된 필지
- `public/data/transport/*_rail_2023.geojson`: 2023-12-31 기준 운영 중인 철도역
- `public/data/analytics/*.json`: 차트와 통계 패널이 참조하는 단일 집계값
- `public/data/reports/*_boundary_match_report.json`: 누락 필지, 중복 필지, 매칭률 로그

## 구현 지표

- 경계 기준: 판교는 성남판교 도시지원시설용지 내 업무연구시설이 집중된 실제 업무 클러스터를 유지하고, 청라는 계획상 국제업무용지만이 아니라 건축물 주용도·연면적, 중심·일반상업지역, 금융업무·하나드림타운 관련 필지, 사업체·종사자 집적을 종합한 실제 업무기능 기반 필지 클러스터로 재정의한다.
- 제1판교테크노밸리 정의: 성남판교 택지개발지구 내 도시지원시설용지 및 업무연구시설이 집중된 제1판교 구역으로 정의한다. 출처는 성남시 지구단위계획 자료 또는 성남판교 도시지원시설용지 고시 자료이다.
- 청라국제업무지구 정의: 청라국제도시 내 실제 업무시설·상업시설 연면적이 크거나 중심·일반상업지역, 금융업무시설, 하나드림타운, 청라국제도시역 주변 업무상업 복합 기능이 확인되는 필지를 업무지구로 정의한다.
- 청라 경계 제외 기준: 순수 공동주택 단지, 학교, 공원, 녹지, 단독주택 위주 지역, 업무시설이 없는 상업주거지역, 음악분수제어실 등 독립 공원·녹지 시설관리용 부속건축물 필지는 제외한다. 단, 업무시설 필지 내부에 존재하는 부속건축물은 필지의 주 기능이 업무·상업이면 유지한다.
- 청라 경계 검증: `validation_report.json`에 포함 필지 수, 포함 건축물 수, 업무시설 연면적, 사업체 수, 종사자 수, 최종 면적을 기록한다.
- 면적 산출: 구역 면적은 경계 GeoJSON을 EPSG:5179 좌표계로 변환한 뒤 산출하였다.
- 인구 지표: 구역 내 집계구 면적비례배분 총인구·가구수, 종사자수·사업체수, 30/60분 통근권 총인구·종사자수
- 산업 지표: SGIS 2023 산업분류 대분류별 사업체수·종사자수 구성
- 건축물 지표: 건축물조서 `건축물대장` 시트의 주건축물을 분석 필지 PNU로 우선 매칭하고, 필요 시 지번으로 보조 매칭한다. 현재 실제 건축물 Polygon이 없으므로 매칭 필지 Polygon을 건축물 대표 Geometry로 사용하며, 추후 건축물 Polygon 데이터가 확보되면 해당 geometry로 교체한다. 건축물 클릭 시 건물명, 주용도, 연면적, 대지면적, 용적률, 건폐율, 층수, 사용승인일, 지역명을 표시한다.
- 토지이용: 토지이용계획 CSV의 `고유번호(PNU)`와 구역 내 분석 필지를 조인해 용도지역지구별 면적 비율을 산출한다.
- 필지 상호작용: 분석 필지 클릭 시 PNU, 지번, 필지면적, 용도지역, 용도지구, 건축물 수, 건축물 연면적 합계, 평균 용적률을 표시한다.
- GIS 컬러맵: 건축물 주용도(업무·상업·주거·교육연구·공공·기타)와 용도지역(일반상업지역·중심상업지역·준주거지역·일반주거지역·도시지원시설용지·기타) 컬러맵을 제공하고, 지도 우측 하단 범례에 구역 경계와 등시간권을 함께 표시한다.
- LUM: `zoning_landuse_mix_index`는 용도지역 면적 기준, `building_landuse_mix_index`는 건축물 주용도별 연면적 기준이다. 건축물 데이터가 정상 매칭되면 대표 `landuse_mix_index`는 건축물 주용도 연면적 기준을 사용하고, 건축물 데이터가 부족하면 용도지역 기준으로 대체한다.
- 교통망: 2023-12-31 기준 운영 중인 철도망에서 최근접 역을 출발점으로 다익스트라 탐색을 수행하고 15·30·45·60분 도달 역의 800m 역세권 집계구 배분값 비교
- 공간접근성: 정류장 밀도, 도로망 밀도, 최근접 고속도로 IC 거리, 500m·1km 역세권 면적 비율을 `spatial_accessibility.json`에 산출하고 KPI 카드, 비교 통계표, 막대그래프에 표시한다.
- 지도 팝업: 필지 클릭 시 PNU·지번 등 속성, 건축물 클릭 시 주용도·건축면적·연면적·대지면적·건폐율·용적률·사용승인일 속성 표출
- 검증 리포트: `public/data/reports/validation_report.json`, `public/data/reports/accessibility_validation_report.json`

## 공간접근성 지표 산식

- 정류장 밀도(개/km²): OSM Overpass API에서 `highway=bus_stop`, `amenity=bus_station` 객체를 조회하고 구역 내부 개수 / 구역면적(km²)으로 산출한다.
- 도로망 밀도(km/km²): OSM Overpass API의 `highway=*` 도로 중심선을 조회하고, 자동차 도로 성격의 `motorway`, `trunk`, `primary`, `secondary`, `tertiary`, `unclassified`, `residential`, `living_street`, `service` 및 각 link 계열을 구역 경계로 clip한 총연장(km) / 구역면적(km²)으로 산출한다.
- 고속도로 IC 접근성(km): OSM Overpass API에서 `highway=motorway_junction` 노드를 조회하고, 구역 중심점에서 최근접 IC까지의 EPSG:5186 거리(km)를 산출한다. 최근접 IC 이름도 함께 기록한다.
- 역세권 면적 비율(%): 2023-12-31 기준 운영 중인 수도권 철도역의 500m 또는 1km 버퍼와 구역 경계의 교차면적 / 구역면적 × 100. 철도역은 `subway_network/network/nodes.tsv`를 사용하고, 2024년 이후 개통·계획 노선은 제외한다.

## 현재 데이터 한계

SGIS 값은 2023년 CSV이고 현재 발견된 집계구 경계는 2025년 2분기 파일입니다. 따라서 값의 기준연도는 2023년, 공간 배분 경계는 사용 가능한 최신 집계구 경계를 활용했습니다. 정류장·도로망·IC 접근성은 OSM Overpass API에서 자동 조회한 최신 공개 공간데이터를 사용합니다. 공식 입주기업 매출·임직원 지표는 아직 없어 가점 지표 일부는 `null`로 둡니다.

대용량 원자료(`서울/서울_토지이용계획/*.csv`, `청라/인천_토지이용계획/*.csv`, `판교/판교_토지이용계획.zip`)는 GitHub 100MB 제한 때문에 `.gitignore`에서 제외합니다. 전처리 결과인 `public/data/analytics/*.json`과 지도용 GeoJSON은 저장소에 포함됩니다.
