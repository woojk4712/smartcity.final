1. 과제 개요

수도권 최고의 성공 사례로 꼽히는 판교테크노밸리와, 조성은 완료되었으나 활성화에 실패(또는 저조)한 업무단지 1곳을 선정하여:

  1. 공공데이터 기반 비교분석 시스템을 직접 구축하고,
  2. 토지이용·교통망·인구사회 지표를 정량적으로 비교한 뒤,
  3. 업무지구의 성공요인을 데이터 근거와 함께 도출하는 보고서를 작성한다.

시스템은 GitHub Pages로 배포하여 누구나 접속 가능해야 하며, 보고서의 모든 주장은 본인이 구축한 시스템에서 확인 가능한 데이터로 뒷받침되어야 한다.

평가의 중심은 분석 내용과 논리적 전개. 화려한 시스템보다, "왜 한 곳은 성공하고 한 곳은 실패했는가"에 대한 데이터 기반의 설득력 있는 답이 도출하고 이를 논리적으로 전개할 수 있도록 한다.

2. 비교 대상 선정

기준 지역 (고정)

  • 판교테크노밸리 (제1판교 기준, 제2판교 포함 여부는 본인이 정의하고 명시할 것)

비교 지역 (선택)

아래 후보군에서 1곳을 선택하거나, 동일한 조건을 만족하는 다른 지역을 직접 발굴해도 좋다.

후보 비고
인천 청라국제업무지구(국제업무단지) 장기 미활성화 논란
위례신도시 업무·상업용지 일대 자족기능 부족 사례로 자주 인용
2기 신도시 자족용지 (운정·양주·김포한강 등 택 1) 미분양·공실 이슈

선정 조건 (반드시 충족)

  • 수도권 내 지역일 것 — 제공되는 지하철 네트워크 데이터(§3-0)의 공간 범위가 수도권이므로, 등시간권 비교가 가능해야 한다
  • 물리적 조성이 완료되어 건축물대장·종사자 데이터가 존재할 것 (사업이 무산된 지역, 예: 용산국제업무지구는 비교할 데이터 자체가 없으므로 불가)
  • "실패(저조)"의 근거를 데이터 또는 공신력 있는 출처(공실률 통계, 미분양 현황, 감사·연구보고서, 언론보도 등)로 보고서에 제시할 것
  • 비교의 공정성을 위해 구역계(경계)를 직접 정의하고, 그 정의의 출처(지구단위계획 고시, 사업지구 경계 등)와 면적을 명시할 것

3. 분석 요구사항 — 필수 지표

세 영역 모두에서 두 지역을 동일한 기준·동일한 기준연도로 비교해야 한다. 아래 필수 지표 외 추가 지표는 자유이며 가점 대상이다.

분석의 범위와 단위를 반드시 명시할 것. 모든 분석에 대해 ① 시간범위 (어느 시점/기간의 데이터인가), ② 공간범위 (어디까지를 분석 대상으로 삼는가), ③ 공간단위 (필지/건축물/집계구 등 무엇을 한 단위로 세는가), ④ 시간단위 (시계열 분석을 할 경우 — 연 단위인가 월 단위인가)를 보고서에 명시해야 한다.

3-0. 지정 데이터

분석에는 아래 4종의 데이터를 반드시 사용한다. (이 외 데이터의 추가 활용은 자유)

데이터 용도 출처
집계구 단위 인구·종사자수 인구사회 분석, 등시간권 도달가능 인구·종사자 산출 SGIS 통계지리정보서비스
토지이용계획(용도지역) + 건축물대장(건축물 용도) 토지이용 분석 VWorld·건축HUB
도로망 구역 내 도로망 지표, 도보 접근 보정 등 OpenStreetMap (OSM)
수도권 지하철 네트워크 그래프 (subway_network.zip) 등시간권 분석 자료 제공 (LMS 배포)

제공 데이터 안내 — subway_network.zip은 수도권 지하철 전 노선(1974년 1호선~GTX 계획 노선)의 역(node)·구간(link) 그래프다. network/nodes.tsv(역별 좌표·노선·개통일)와 network/links.tsv(fromNode, toNode, 방향별 소요시간 초 단위)로 구성되며, 환승 링크에는 배차간격 기반 대기시간이 이미 반영되어 있다. 즉 그래프를 직접 만들 필요 없이, 최단시간 탐색(다익스트라)부터 시작하면 된다. 자세한 스키마와 예제 코드는 zip 안의 README 참고. begin/effective_begin 컬럼으로 특정 시점의 운영 네트워크만 골라낼 수 있다 (조성 전후 비교 등 시계열 분석 가능).

3-1. 토지이용 (Land Use)

지표 정의 / 산출 방법 데이터 출처
용도지역 구성비 구역 내 용도지역별 면적 비율 VWorld 토지이용계획정보
건축물 주용도 구성비 주용도별 연면적(또는 필지 수) 비율 — 업무·주거·근생·교육연구 등 건축물대장 (건축HUB)
토지이용 혼합도 엔트로피 지수(LUM) 등 혼합도 지표 1개 이상 위 데이터 가공
개발 실현 정도 평균 용적률(연면적/대지면적) 또는 공지(미건축 필지) 비율 건축물대장 + 연속지적도

3-2. 교통망 (Transport) — 등시간권 분석 중심

교통망 분석의 핵심은 업무지구 중심부의 대중교통 접근성이다. 각 지역의 핵심역(중심 지하철·광역철도역) 1개를 출발점으로 정의하고, 다음을 산출·표출한다.

핵심 분석: 등시간권 (Isochrone)

지표 정의 / 산출 방법 데이터 출처 (예시)
핵심역 정의 업무지구 중심부를 대표하는 역 선정 + 선정 근거 (예: 판교역 vs ____역) 본인 정의
30분·60분 등시간권 핵심역에서 지하철로 30분 / 60분 이내 도달 가능한 공간 범위 — 지도에 폴리곤으로 표출 제공: subway_network.zip (§3-0)
도달가능 인구·종사자 각 등시간권 안의 총인구와 총종사자수 — 두 지역을 동일 시간대로 비교 등시간권 × SGIS 집계구 (인구·종사자) 공간 결합
누적 접근성 곡선 (권장) 소요시간 t분 이내 도달가능 인구·종사자를 t에 대한 곡선으로 표현 (예: 0~60분) 위 데이터 가공

해석 포인트: "핵심역에서 30분 안에 닿는 종사자(또는 경제활동인구)가 몇 명인가"는 그 업무지구가 끌어올 수 있는 노동시장의 크기를 의미한다. 두 지역의 격차가 어디서(어느 시간대, 어느 방향에서) 벌어지는지를 읽어낼 것.

보조 지표 (1개 이상 선택)

지표 정의 / 산출 방법 데이터 출처 (예시)
역세권 면적 비율 구역 내 역세권(500m/1km) 면적 비율 subway_network.zip 역 좌표 활용
버스 접근성 구역 내 정류장 밀도(개/km²) 국토교통부 정류장 정보
도로망 수준 도로율·도로망 밀도(km/km²), 고속도로 IC까지 거리 OSM

3-3. 인구사회 (Socio-demographics)

지표 정의 / 산출 방법 데이터 출처
인구·가구 구역 및 주변(통근권) 집계구 총인구·가구수 SGIS 집계구 통계
종사자·사업체 구역 내 종사자수·사업체수, 업종(산업분류) 구성 SGIS 전국사업체조사
직주 지표 직주비(종사자수/상주인구) 등 1개 이상 위 데이터 가공

공간 단위 주의: 데이터마다 단위(필지/건축물/집계구)가 다르다. 구역계와 집계구 경계가 일치하지 않을 때 어떻게 처리했는지(포함 기준, 면적 비례 배분 등)를 보고서에 반드시 명시할 것 — 8강에서 다룬 "공간단위 통합" 문제다.

4. 시스템 요구사항

8강 실습(송파구 데이터 뷰어)의 구조를 두 지역 비교형으로 확장한다.

필수 기능

  1. 지도 기반 시각화: 두 구역의 경계 + 필지/건축물 단위 주용도(또는 용도지역) 컬러맵. 두 지역을 전환하거나 나란히(side-by-side) 볼 수 있을 것
  2. 등시간권 레이어: 핵심역 기준 30분/60분 등시간권 폴리곤을 지도에 표출하고, 시간대를 전환(또는 슬라이더 조절)할 수 있을 것. 각 등시간권의 도달가능 인구·종사자 수치가 함께 표시될 것
  3. 통계 패널: 영역별(3-1~3-3) 비교 통계표와 차트(파이/막대, 누적 접근성 곡선 등) — 두 지역 수치가 한 화면에서 비교될 것
  4. 상호작용: 필지/건축물 클릭 시 속성(주용도·연면적·용적률 등) 표출

기술 조건

  • GitHub Pages 배포 (정적 사이트) — 별도 서버·DB 없이 동작해야 하므로, 데이터는 전처리하여 GeoJSON/JSON 등 정적 파일로 포함할 것
  • 베이스맵: Vworld WMTS, OSM 등 자유
  • 데이터 전처리 스크립트(Python 등)를 저장소에 포함하고, README에 데이터 출처·기준월·처리 과정을 기록할 것 (재현 가능성)
  • AI 코딩 도구(바이브코딩) 적극 활용 권장 — 단, 결과물의 정확성 검증 책임은 본인에게 있다

5. 보고서 요구사항

분량 A4 10매 이하 (그림·표 포함, 표지 및 참고문헌 페이지는 분량에서 제외, 11pt 기준). 구성:
분량 조절이 어려울 경우 부록 섹션을 만들어서 부록에 부수적인 분석이나 내용을 추가하는 것은 가능함

  1. 서론: 문제의식, 비교 지역 선정 근거 (실패 판단의 데이터·출처 포함)
  2. 분석 방법: 구역계 정의, 데이터 출처, 분석의 시간범위·공간범위·공간단위·시간단위(시계열 분석 시), 공간단위 통합 방법
  3. 비교 분석: 토지이용 / 교통망 / 인구사회 — 각 영역별 정량 비교 결과와 해석
  4. 성공요인 도출: 세 영역의 결과를 종합하여 업무지구 성공요인을 3가지 이상 제시. 각 요인은 본인의 분석 결과(수치)로 뒷받침할 것
  5. 분석의 한계: 상관과 인과의 구분, 데이터·방법의 한계를 스스로 서술
  6. 부록: 시스템 URL, 저장소 URL, 데이터 출처 목록, AI 활용 내역

작성 원칙 — 좋은 보고서는 ① 간결하고 이해하기 쉽다 (긴 보고서가 아니라 핵심이 분명한 보고서), ② 논리적으로 전개된다 (문제의식부터 성공요인까지 하나의 논증), ③ 시스템과 연결된다 (보고서의 모든 핵심 수치·그림은 배포된 시스템에서 확인 가능해야 하며, 어느 화면을 근거로 삼는지 명시). 평가 기준(§7)이 이 세 가지를 직접 채점한다.

감점 사항

아래의 경우 정도에 따라 감점한다.

  • 데이터 출처·기준연도 미표기, 또는 두 지역을 서로 다른 기준으로 비교한 경우
  • 정돈되지 않은 보고서 (문단과 문장의 구분이 없는 글, 섹션 구성의 부재, 표·그림 번호와 본문 참조의 불일치 등)
  • 보고서의 수치와 시스템에 표출되는 수치가 일치하지 않는 경우
  • 시스템이 배포되지 않았거나 (로컬에서만 동작) 정상 동작하지 않는 경우

기한 외 제출

  • 마감 이후 제출 시 24시간 단위로 10점씩 감점한다 (마감 후 24시간 이내 −10점, 48시간 이내 −20점, 72시간 이내 −30점).
  • 마감 후 72시간을 초과하면 미제출(0점)로 처리한다.
  • 제출 시각은 LMS 기록을 기준으로 하며, 시스템(GitHub Pages)·저장소는 마감 시점의 상태로 평가한다 (마감 이후 커밋은 평가에 반영하지 않음).

8. AI 활용 정책

  • 코드 작성, 데이터 처리, 문장 다듬기에 AI 도구를 적극 활용해도 좋다 (수업에서 권장한 방식이다).
  • 단, AI가 생성한 수치·사실·법령·출처는 반드시 원데이터로 검증할 것.
  • 보고서 부록에 AI 활용 내역(도구명, 활용 범위)을 한 문단으로 기술할 것. 활용 자체는 감점 사유가 아니다.
  • 분석의 해석과 결론은 본인의 것이어야 하며, 보고서를 쓸 때는 특히 주의(지나친 약어 사용, AI가 생성한 문장을 수정없이 그대로 사용 등)
